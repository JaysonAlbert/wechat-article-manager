# 微信公众号 Markdown 写作最佳实践

## Frontmatter 规范

文章头部必须包含 frontmatter，格式如下：

```yaml
---
title: 文章标题
date: 2026-04-06
tags: 标签 1, 标签 2
---
```

- **title**: 必填，文章标题
- **date**: 必填，发布日期 (YYYY-MM-DD)
- **tags**: 可选，逗号分隔的标签列表

## 标题使用

- 使用 `#` 表示一级标题，`##` 表示二级标题
- 公众号仅支持这两级标题
- `#` 后需要加一个空格

```markdown
# 这是标题

## 这是副标题

### 这在公众号不会显示
```

## 代码块

代码块在公众号渲染良好，支持语法高亮：

```javascript
console.log('Hello, World!');
```

建议使用反引号或代码块语法，避免手动缩进。

## 样式格式

### 粗体和斜体

```markdown
**粗体文本**
_斜体文本_
***粗斜体***
```

### 引用文本

```markdown
> 引用内容，重要信息突出显示
```

### 列表

```markdown
- 无序列表项
- 又一项目

1. 有序列表
2. 第二项
```

### 链接和图片

```markdown
[链接文本](https://example.com)

![图片描述](图片 URL)
```

## 排版建议

- **控制段落间距**: 空行分隔段落，增强可读性
- **避免标题嵌套过深**: 公众号不支持多级标题
- **代码块宽度**: 代码块会自适应宽度，便于阅读
- **保持简洁**: 避免过度使用格式，聚焦内容

## 主题风格

发布时可选主题决定整体样式：

| 主题 | 特点 |
|------|------|
| `default` | 经典布局，适合长文 |
| `orangeheart` | 暖橙色，优雅明亮 |
| `lapis` | 蓝色调，清新简约 |
| `purple` | 紫色点缀，极简风格 |
| `pie` | 仿 sspai 风格，现代时尚 |

## 注意事项

1. 图片建议使用外部图床，避免本地图片
2. 发布前在编辑器预览效果
3. 代码块支持多种语言标识
4. 特殊符号可能需要转义
